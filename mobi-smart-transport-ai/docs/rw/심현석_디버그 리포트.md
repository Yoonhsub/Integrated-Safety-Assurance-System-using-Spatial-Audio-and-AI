# 심현석_디버그 리포트

> 이 문서는 심현석 담당 AI 에이전트가 섹션 3, 5, 7, 9, 11의 **검토/패치 단계**에서 반드시 최신화해야 하는 디버그 리포트이다.  
> 이 문서는 실제 개발 완료 보고서가 아니라, 각 검토 단계에서 발견한 문제·패치 내역·미해결 이슈를 추적하기 위한 문서이다.

---

## 1. 담당 범위

- 담당 팀원: 심현석
- 담당 역할: FastAPI 백엔드, Firebase Realtime Database, 지오펜싱, FCM, 기사-승객 매칭 데이터 파이프라인
- 주요 소유 경로: `backend/api, infrastructure/firebase, packages/shared_contracts 중 백엔드 계약 관련 영역`
- 직접 담당하지 않는 범위: 공공데이터 API 직접 구현, Flutter UI, BLE/RSSI, AI 비전, 헤드트래킹

---

## 2. 작성 원칙

```txt
1. 섹션 3, 5, 7, 9, 11이 끝날 때마다 반드시 최신화한다.
2. 이전 구현 섹션의 결과물을 먼저 검토한다.
3. 이상이 있으면 해당 부분만 패치한다.
4. 이상이 없으면 "다음 섹션으로 넘어가도 된다"고 사용자에게 보고한다.
5. 다음 섹션으로 임의 점프하지 않는다.
6. 충돌 가능성이 있으면 docs/rw/충돌 이슈.md에 기록하고 개발을 중단한다.
7. docs/rw/공통 진행사항.md의 자기 팀원 기록 공간도 함께 최신화한다.
8. 다른 팀원의 디버그 리포트를 수정하지 않는다.
```

## 6. 섹션별 검토 기준

### 섹션 3 검토 기준

- 섹션 2 구현 결과가 담당 범위와 일치하는지 확인한다.
- 기본 프로젝트 구조, 초기 설정, 인터페이스가 깨지지 않았는지 확인한다.
- 다른 팀원 소유 경로를 침범하지 않았는지 확인한다.

### 섹션 5 검토 기준

- 섹션 4 구현 결과의 주요 로직을 검토한다.
- API, UI, 센서, 데이터 표준화 등 담당 영역별 핵심 기능이 문서와 일치하는지 확인한다.
- 발견된 문제는 가능한 한 해당 담당 영역 내부에서만 패치한다.

### 섹션 7 검토 기준

- 섹션 6 구현 결과의 예외 처리, 상태값, mock/테스트 흐름을 검토한다.
- 충돌 가능성이 있는 인터페이스 변경이 있었는지 확인한다.
- 필요한 경우 docs/rw/충돌 이슈.md에 기록한다.

### 섹션 9 검토 기준

- 섹션 8 구현 결과가 다른 모듈과 연동 가능한 형태인지 확인한다.
- API 계약, JSON 필드명, 상태값, 파일 경계가 문서와 일치하는지 확인한다.

### 섹션 11 검토 기준

- 섹션 10 구현 결과를 최종 구현 전 단계 수준으로 검토한다.
- 섹션 12 최종 심층검증에 들어가기 전에 남은 문제를 정리한다.
- 미해결 이슈가 있으면 명확히 표시한다.
