# 안준환_디버그 리포트

> 이 문서는 안준환 담당 AI 에이전트가 섹션 3, 5, 7, 9, 11의 **검토/패치 단계**에서 반드시 최신화해야 하는 디버그 리포트이다.  
> 이 문서는 실제 개발 완료 보고서가 아니라, 각 검토 단계에서 발견한 문제·패치 내역·미해결 이슈를 추적하기 위한 문서이다.

---

## 1. 담당 범위

- 담당 팀원: 안준환
- 담당 역할: 스마트폰 방향 센서, BLE 비콘 수신, RSSI 기반 거리 추정, 모바일 센서 모듈화
- 주요 소유 경로: `packages/mobile_sensors`
- 직접 담당하지 않는 범위: 4월 헤드트래킹 센서 통신, 헤드트래킹 기반 공간음향, Flutter UI, FastAPI, Firebase, 공공데이터 API, AI 비전

---

## 2. 작성 원칙

```txt
1. 섹션 3, 5, 7, 9, 11이 끝날 때마다 반드시 최신화한다.
2. 이전 구현 섹션의 결과물을 먼저 검토한다.
3. 이상이 있으면 해당 부분만 패치한다.
4. 이상이 없으면 "다음 섹션으로 넘어가도 된다"고 사용자에게 보고한다.
5. 다음 섹션으로 임의 점프하지 않는다.
6. 충돌 가능성이 있으면 docs/rw/충돌 이슈.md에 기록하고 개발을 중단한다.
7. docs/rw/공통 진행사항.md의 자기 팀원 기록 공간도 함께 최신화한다.
8. 다른 팀원의 디버그 리포트를 수정하지 않는다.
```
