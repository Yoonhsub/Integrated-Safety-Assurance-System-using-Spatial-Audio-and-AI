# 김도성_디버그 리포트

> 이 문서는 김도성 담당 AI 에이전트가 섹션 3, 5, 7, 9, 11의 **검토/패치 단계**에서 반드시 최신화해야 하는 디버그 리포트이다.  
> 이 문서는 실제 개발 완료 보고서가 아니라, 각 검토 단계에서 발견한 문제·패치 내역·미해결 이슈를 추적하기 위한 문서이다.

---

## 1. 담당 범위

- 담당 팀원: 김도성
- 담당 역할: 공공데이터 API 기반 버스 정보 연동, 저상버스 필터링, 표준 응답 JSON/mock 데이터, AI 비전 데이터 수집 계획 및 모델 리서치
- 주요 소유 경로: `services/public_data, ai_vision`
- 직접 담당하지 않는 범위: 비콘 거리 테스트 계획, 공간음향 방향 테스트 계획, FastAPI 백엔드, Firebase, FCM, Flutter UI, BLE/RSSI, 헤드트래킹


## 6. 섹션별 검토 기준

### 섹션 3 검토 기준

- 섹션 2 구현 결과가 담당 범위와 일치하는지 확인한다.

- 다른 팀원 소유 경로를 침범하지 않았는지 확인한다.

### 섹션 5 검토 기준

- 섹션 4 구현 결과의 주요 로직을 검토한다.

- 발견된 문제는 가능한 한 해당 담당 영역 내부에서만 패치한다.

### 섹션 7 검토 기준

- 섹션 6 구현 결과의 예외 처리, 상태값, mock/테스트 흐름을 검토한다.
