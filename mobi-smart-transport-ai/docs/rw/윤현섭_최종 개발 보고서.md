# 윤현섭 최종 개발 보고서

## 1. 담당 범위

윤현섭 담당 범위는 다음 앱 UI 영역으로 한정하였다.

- `apps/passenger_app/**`
- `apps/driver_app/**`

이번 작업에서는 승객 앱과 기사 앱의 Flutter UI를 중심으로 개발하였다.  
백엔드 API, Firebase, 공공데이터 수집 모듈, BLE/RSSI 모듈, AI Vision, shared contracts 영역은 임의로 수정하지 않았다.

수정하지 않은 범위는 다음과 같다.

- `backend/api/**`
- `infrastructure/firebase/**`
- `services/public_data/**`
- `packages/mobile_sensors/**`
- `ai_vision/**`
- `packages/shared_contracts/**`

## 4. 미구현 기능 및 후속 작업 필요 사항

### 4.1 실제 백엔드 API 연동

현재 승객 앱의 안전 상태, 버스 도착 정보, 탑승 요청 상태는 mock 또는 연결 예정 UI 중심으로 구현하였다.

후속 작업이 필요한 API는 다음과 같다.

- `/geofence/check`
- `/bus-info/stops/{stopId}/arrivals`
- `/ride-requests`

실제 API 계약이 확정되면 `BackendApiClient`에서 해당 API를 호출하고, 응답값을 홈 화면 카드에 렌더링하도록 확장해야 한다.

### 4.2 실제 rideRequests 처리

기사 앱은 현재 탑승 요청 UI shell과 정적 예시 카드만 구현되어 있다.

후속 작업이 필요한 항목은 다음과 같다.

- 실제 탑승 요청 목록 조회
- 탑승 요청 상세 표시
- 탑승 요청 확인 또는 처리 액션
- rideRequests status 처리
- 실시간 요청 업데이트

단, 이 항목들은 rideRequests 파이프라인과 schema가 확정된 후 구현해야 한다.

### 4.3 실제 STT/TTS 고도화

승객 앱에는 음성 입력 및 안내 흐름을 위한 wrapper 구조를 구현하였다.

후속 작업이 필요한 항목은 다음과 같다.

- 실제 STT 인식 정확도 개선
- 실제 TTS 음성 출력 품질 조정
- 음성 권한 예외 처리
- 음성 인식 실패 처리
- 목적지 인식 결과와 경로/탑승 요청 흐름 연결

### 4.4 접근성 실기기 테스트

현재 Flutter `Semantics`를 적용했지만, 실제 Android TalkBack 또는 iOS VoiceOver 실기기 검증은 추가로 필요하다.

후속 작업이 필요한 항목은 다음과 같다.

- TalkBack 포커스 순서 확인
- VoiceOver 읽기 순서 확인
- 큰 글씨 환경 확인
- 고대비 모드 확인
- 터치 영역 크기 확인

## 9. PR 반영 대기 중인 추가 수정 사항

최종 보고서 작성 시점 기준으로, 기존 리뷰 피드백 중 일부 항목은 코드 수정 및 커밋까지 완료했으나 아직 메인 브랜치에는 병합되지 않은 상태이다. 따라서 해당 항목들은 PR 리뷰 및 merge 이후 메인 브랜치에 최종 반영될 예정이다.

### 9.1 STT/TTS 실제 연동 보강

기존 `VoiceGuideService`는 실제 STT/TTS 플러그인이 연결되지 않은 stub 구조였다.  
리뷰 피드백에 따라 `speech_to_text`, `flutter_tts` 기반의 실제 음성 인식 및 음성 안내 연동을 추가하였다.

수정 내용은 다음과 같다.

- `speech_to_text` 의존성 추가
- `flutter_tts` 의존성 추가
- `VoiceGuideService`를 실제 STT/TTS plugin wrapper 구조로 수정
- 음성 입력 시작 시 `speech_to_text.listen()`을 통해 음성 인식 시작
- 인식 결과를 승객 앱 음성 안내 상태 카드에 반영
- 안내 문구를 `flutter_tts.speak()`로 출력하도록 연결
- 마이크 권한 또는 음성 인식 사용 불가 상황에 대한 안내 문구 추가

관련 수정 파일은 다음과 같다.

- `apps/passenger_app/pubspec.yaml`
- `apps/passenger_app/pubspec.lock`
- `apps/passenger_app/lib/src/services/voice_guide_service.dart`
- `apps/passenger_app/lib/src/pages/home_page.dart`

현재 상태:

- 코드 수정 완료
- 커밋 완료
- 메인 브랜치 병합 전
- PR 리뷰 및 merge 후 최종 반영 예정

### 9.2 승객 앱 백엔드/mock 상태 렌더링 보강

기존 승객 앱 홈 화면의 안전 상태, 버스 도착 정보, 탑승 요청 상태 카드는 고정 placeholder 문구 중심으로 표시되었다.  
리뷰 피드백에 따라 `BackendApiClient`를 실제로 사용하도록 연결하고, mock 응답값이 카드에 렌더링되도록 개선하였다.

수정 내용은 다음과 같다.

- `BackendApiClient`에 승객 앱 홈 화면용 mock snapshot 반환 메서드 추가
- 안전 상태 mock 값 렌더링
- 버스 도착 정보 mock 값 렌더링
- 탑승 요청 상태 mock 값 렌더링
- `home_page.dart`에서 `BackendApiClient`를 호출해 카드 UI에 반영
- API/DB/schema/status enum은 새로 정의하지 않고 mock 기반 렌더링으로 처리

관련 수정 파일은 다음과 같다.

- `apps/passenger_app/lib/src/services/backend_api_client.dart`
- `apps/passenger_app/lib/src/pages/home_page.dart`

현재 상태:

- 코드 수정 완료
- 커밋 완료
- 메인 브랜치 병합 전
- PR 리뷰 및 merge 후 최종 반영 예정

### 9.3 최종 반영 전 유의 사항
