## 2. 개발 전 절대 확인 사항

AI 에이전트는 개발 시작 전에 반드시 다음을 확인한다.

```txt
1. 나는 어느 팀원의 에이전트인가?
2. 내가 수정하려는 파일이 그 팀원의 담당 영역인가?
3. 요구사항명세서와 4월 구현범위 수정안에 내 작업이 포함되어 있는가?
4. 다른 팀원의 담당 영역을 수정해야만 하는가?
5. 수정이 충돌 가능성을 만들 경우, 개발을 멈추고 충돌 이슈 기록이 필요한가?
```

위 질문 중 1번이 불명확하면 **절대 개발을 시작하지 않는다.**

## 3. 이 패키지의 성격

이 저장소는 완성 기능 구현물이 아니라, 팀원 4명이 생성형 AI를 분산 활용해도 충돌을 최소화할 수 있도록 만든 **공통 프로젝트 아키텍처 스캐폴딩**이다.

따라서 현재 코드는 다음 원칙을 따른다.

```txt
- 세부 기능 완성보다 모듈 경계와 계약을 우선한다.
- 4월 구현 범위는 각 팀원 담당 디렉터리 안에서만 진행한다.
- 지금 당장 구현하지 않을 헤드트래킹/공간음향 등도 향후 확장을 위해 프레임만 둔다.
- 공공데이터 API는 김도성 담당 영역으로 분리되어 있다.
- FastAPI/Firebase/지오펜싱/FCM/매칭은 심현석 담당 영역으로 분리되어 있다.
- BLE/RSSI/스마트폰 방향 센서는 안준환 담당 영역으로 분리되어 있다.
- Flutter 사용자 앱/기사 앱 UI는 윤현섭 담당 영역으로 분리되어 있다.
```

## 4. 문서 원본 보존 원칙

`docs/agent_required_reading/` 내부 문서는 프로젝트 기준 문서이므로, 일반 개발 에이전트가 임의로 수정하지 않는다.

문서 변경이 필요한 경우에는 별도 변경 이유를 남기고, 팀원 간 합의 후 수정한다.

## 추가 필독 문서: docs/rw/선행작업의존성 정리.md

모든 AI 에이전트는 섹션 1을 시작하기 전에 프로젝트의 `docs/rw/선행작업의존성 정리.md`를 반드시 읽어야 한다.

이 문서는 다음을 정의한다.

```txt
- 4월 구현범위 내 강한 선행 작업 의존성
- 이후 구현범위의 잠재적 선행 작업 의존성
- 선행 섹션이 (미구현)일 때 후행 에이전트가 중단해야 하는 기준
- 선행 섹션이 (구현완료_날짜)일 때 실제 구현물 2차 확인 절차
- 선행 섹션 구현 완료 후 상태 최신화 규칙
```

특히 후행 에이전트는 선행 작업물이 없을 때 임의로 mock/API/DB 구조를 추정해서 구현해서는 안 된다.
