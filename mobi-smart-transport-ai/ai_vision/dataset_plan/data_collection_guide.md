# 데이터 수집 가이드 (Dataset Plan / Data Collection)

이 문서는 MOBI 프로젝트 2학기 AI 비전 기능을 위한 데이터 수집 기준이다.
4월 단계에서는 **수집 계획 문서화**가 목표이며, 실제 대규모 수집은 2학기에 본격 수행한다.

---

## 1. 수집 목적

`class_taxonomy.json`에 정의된 7개 클래스(`bus`, `bus_door`, `bus_stop`, `roadway`,
`sidewalk`, `obstacle`, `tactile_paving`)를 모바일 경량 모델(YOLOv8n / YOLOv11n /
EfficientDet-Lite0 등 — `model_research/`)이 인식하도록 학습할 데이터를 준비한다.

수집 데이터의 최종 사용처는 다음과 같다.

- **2학기 본구현**의 학습/평가 데이터셋
- 4월 단계에서는 학습/추론 코드를 작성하지 않으므로, 본 가이드는 **수집 시점의
  품질 기준**을 미리 정의해 두는 데 목적이 있다.

---

## 2. 수집 매체

### 2.1 1차 권장 — 직접 촬영

- 스마트폰 후면 카메라 (사용자가 보행 시 들고 다닐 위치/높이를 모사)
- 권장 해상도: **1920×1080 이상** (학습 단계에서 640×640으로 down-sample 가능)
- 권장 프레임레이트: 30 fps (정지 이미지 추출용으로 충분)
- 가로 세로 비율: 16:9 표준 (변경 시 메타데이터에 기록)

### 2.2 2차 보조 — 공개 데이터셋 활용

다음 공개 데이터셋의 이미지를 보조로 사용할 수 있다. 다만 **타깃 도시(청주·대전 등
중부권)의 정류장 환경**과 다를 수 있어 주력 데이터로는 부적합.

- COCO (`bus`, `traffic_light` 등 일반 클래스만 매칭)
- BDD100K, Cityscapes (자율주행 도메인 — `roadway`, `sidewalk`)
- 한국 AI Hub의 보행자 안전 관련 공개 데이터셋 (가용성·라이선스는 2학기 시작 시점에 재확인)

공개 데이터셋 사용 시 **반드시 라이선스 호환 확인**을 먼저 하고, 문서에 출처와
라이선스 종류를 기록한다.

### 2.3 4월 단계의 보조 활동

4월에는 본격 수집 대신 다음을 권장한다.

- 팀원 1인당 정류장 사진 10~30장 시범 촬영 (본 가이드 검증 목적)
- 시범 촬영 결과로 본 문서의 미흡한 부분을 보강하는 회고 1회

## 3. 촬영 환경 분포

학습 데이터의 환경 다양성은 모델 일반화에 직결된다. 다음 차원을 균형 있게 분포시킨다.

### 3.1 시간대

```txt
- 주간 맑음    : 30%
- 주간 흐림/비  : 25%
- 일출/일몰   : 15%
- 야간 (가로등) : 30%
```

야간 데이터를 30%까지 확보하는 이유: 교통약자가 야간에 이동할 때 안전 안내가 더
중요하지만, 모바일 카메라의 야간 노이즈/흐림으로 인해 모델 성능이 가장 떨어지는
구간이기 때문이다.

### 3.2 정류장 형태

```txt
- 표지판형 (지붕 없음)        : 40%
- 승강장형 (지붕·벤치)        : 40%
- 환승센터/터미널 인접형      : 10%
- 임시 정류장/공사 우회       : 10%
```

### 3.3 카메라 시점

```txt
- 보행자 시선(170 cm 높이 기준)     : 70%
- 휠체어 사용자 시선(120 cm 높이)   : 20%
- 어린이/저신장 시선(100 cm 높이)   : 10%
```

휠체어/저신장 시점은 같은 정류장이라도 표지판이 잘 안 보이는 사각이 다르므로 별도 확보.

### 3.4 거리

```txt
- 근거리 (정류장 직전, 1~3 m)        : 40%
- 중거리 (5~10 m, 접근 중)           : 40%
- 원거리 (10~30 m, 인지 단계)        : 20%
```

## 4. 위치 정보·개인정보 보호

### 4.1 GPS 좌표 처리

촬영 이미지의 EXIF에 GPS 좌표가 포함되어 있을 수 있다. **데이터셋 빌드 단계에서
좌표는 다음 중 하나의 방식으로 익명화한다.**

```txt
- 옵션 A: EXIF 자체를 모두 제거 (가장 단순, 권장)
- 옵션 B: 좌표를 km 격자로 양자화 (서울/청주 단위로 충분)
- 옵션 C: 좌표 메타데이터를 별도 보안 저장소에 보관하고, 학습 데이터에서는 제거
```

본 프로젝트는 **옵션 A를 기본**으로 한다. 위치 메타가 학습에 직접 쓰이지 않기 때문이다.

### 4.2 사람 얼굴

`class_taxonomy.json`의 `non_classes`에 명시한 대로, 사람을 학습 클래스로 두지 않으며
**얼굴은 블러 처리한다**. 처리 방식:

```txt
- 자동: 일반적인 face detection으로 얼굴 영역 탐지 후 Gaussian blur 적용
- 수동: 자동 결과 확인 후 누락된 얼굴은 어노테이터가 추가 블러
- 어린이 얼굴은 더 큰 마진으로 블러 (자동 face detection이 어린이 얼굴을 덜 잡는 경향)
```

블러 처리는 **라벨링 직전 단계**에서 일괄 수행한다. 어노테이터는 블러 처리된 이미지만 본다.

### 4.3 차량 번호판

`non_classes`에 명시한 대로 번호판은 블러 처리한다. 자동 OCR/Detector가 번호판을
잡지 못한 경우 어노테이터가 수동으로 마킹한다.

### 4.4 개인 식별 가능 정보 일반

상점 간판, 명함, 메뉴판, 가게 외벽의 개인 이름 등도 가능하면 회피하여 촬영한다.
회피하지 못한 경우 후처리 단계에서 부분 블러 적용.

## 5. 수집 양·품질 기준

### 5.1 4월 시범 단계 (현재)

```txt
- 팀 전체 50~150장
- 환경 분포 검증용 (3.1~3.4 차원이 실제로 균형 있게 모이는지 확인)
- 라벨링 가이드 미흡한 부분 발굴
```

### 5.2 2학기 본격 수집 목표 (참고치)

```txt
- 클래스당 최소 500장 (priority=high) / 200장 (priority=medium)
- 7개 클래스 합산 약 3000~5000장 (이미지 기준)
- 학습/검증/테스트 분할 = 70 / 15 / 15
- 정류장 위치는 최소 10개소 이상 분산
```

위 수치는 4월 시점의 추정치이며, 2학기 시작 시점에 실험 결과를 보고 조정한다.

### 5.3 품질 기준 (수집 단계 통과 조건)

수집된 이미지는 다음 기준을 만족해야 데이터셋에 포함된다.

```txt
- 초점이 맞고 모션 블러가 심하지 않음 (육안 식별 가능)
- 해상도 1280x720 이상 (저화질 폰 보정용 fallback 임계값)
- 7개 클래스 중 최소 1개가 이미지 안에 포함
- EXIF GPS 제거 또는 양자화 완료
- 사람 얼굴/번호판 블러 처리 완료
```

## 6. 메타데이터 규약

각 이미지에 다음 메타를 함께 기록한다 (별도 JSON 또는 CSV로 관리; 4월 단계에서는 양식만 합의).

```txt
- image_id            : 고유 식별자 (UUID 또는 순번)
- captured_at         : 촬영일자 (분 단위까지)
- time_of_day_bucket  : day_clear / day_overcast / sunrise_sunset / night
- stop_type           : sign_only / shelter / hub / temporary
- camera_height_bucket: 170cm / 120cm / 100cm
- distance_bucket     : near / mid / far
- city                : seoul / cheongju / daejeon / ...
- weather             : clear / rain / snow / overcast
- pii_blur_done       : true / false
- gps_anon_done       : true / false
- collector_name      : 수집자 이름 (팀 내부 식별, 외부 공개 안 함)
- license             : 4월 시범 수집은 internal_only 고정. 2학기 공개 시 별도 결정.
```
