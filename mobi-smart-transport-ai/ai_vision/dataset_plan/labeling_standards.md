# 라벨링 표준 (Dataset Plan / Labeling Standards)

이 문서는 MOBI 프로젝트 2학기 AI 비전 학습 데이터의 **어노테이션 일관성**을 위한
라벨링 기준이다. 4월 단계에서는 기준 정의가 목표이며, 실제 대규모 라벨링은
2학기에 수행한다.

`class_taxonomy.json`과 함께 읽는다 — taxonomy는 **무엇을 라벨링하는가**, 본 문서는
**어떻게 라벨링하는가**를 다룬다.

---

## 1. 어노테이션 형식

기본 형식: **bbox** (axis-aligned bounding box).
좌표 표기는 학습 프레임워크 표준에 맞춰 다음 두 형식 모두를 지원할 수 있도록 저장.

```txt
- YOLO 형식: <class_id> <x_center_norm> <y_center_norm> <w_norm> <h_norm>
  - 좌표/크기는 이미지 가로·세로로 0~1 정규화
- COCO 형식: [x, y, width, height] (절대 픽셀)
```

4월 단계에서는 형식 결정만 하고 실제 학습 변환은 2학기에 수행한다.
Segmentation은 4월 범위에서 수행하지 않는다 (`class_taxonomy.json` 참조).

## 2. bbox 그리는 일반 규칙

### 2.1 타이트하게

```txt
- bbox는 객체의 가장자리에 정확히 붙도록 그린다.
- 객체의 그림자, 반사, 광원 효과는 bbox 밖으로 둔다.
- 너무 타이트하게 그려서 객체의 일부가 잘리는 것보다, 1~2 픽셀 여유를 두는 편이 안전.
```

### 2.2 일부 가려진 객체 (occlusion)

```txt
- 객체의 50% 이상이 보이면 → 보이는 부분만 bbox로 라벨링.
- 객체의 50% 미만만 보이면 → 라벨링 제외.
- 어노테이터가 모호하다고 느끼면 라벨링 제외 후 미정 케이스 보고 절차로 보낸다 (§5).
```

### 2.3 잘린 객체 (truncation, 이미지 경계에 걸침)

```txt
- 이미지 경계에 의해 객체가 잘린 경우, 이미지 안쪽 부분만 bbox로 그린다.
- bbox가 이미지 경계를 넘어가지 않게 하고, 잘림 사실은 메타에 기록 (truncated=true).
- 매우 작게 잘려서 식별이 어려우면 라벨링 제외.
```

### 2.4 동일 객체가 여러 부분으로 보이는 경우

```txt
- 한 객체가 가림에 의해 둘로 보여도 단일 bbox로 그린다 (외접 사각형).
- 단 너무 멀리 떨어진 두 부분(예: 가림 객체가 매우 큰 경우)은 어노테이터 판단으로
  분할하지 않고, 일관성을 위해 §5 미정 케이스로 보고한다.
```

### 2.5 매우 작은 객체

```txt
- 가로/세로 어느 한쪽이 16 픽셀 미만인 객체는 라벨링 제외.
- 모바일 모델(YOLO/EfficientDet-Lite)이 작은 객체에 약하기 때문에 학습 노이즈만 늘림.
```

## 3. 클래스별 세부 규칙

### 3.1 `bus`

- 차체 외곽 bbox.
- 양 옆이 잘린 측면 차체는 라벨링 가능 (§2.3).
- 트럭/택시/SUV는 본 클래스에 포함하지 않는다 (혼동 시 미정 케이스 보고).
- 노란색 학교 버스는 본 클래스에 포함 (4월 단계 단일 처리).

### 3.2 `bus_door`

- 버스 차체 안의 문 영역만 라벨링.
- 앞문/중문 모두 동일 클래스. 비상문(후방)은 제외.
- 닫힘/열림 구분하지 않음. 손잡이/계단까지는 bbox에 포함.
- `bus`와 중첩하여 라벨링한다 (즉, 한 픽셀이 동시에 `bus`와 `bus_door` 둘 다일 수 있음).

### 3.3 `bus_stop`

- 표지판 + 인접한 승강장 구조물(지붕, 벤치, 안내판)을 외접 bbox.
- 표지판이 가려져 식별 불가하면 라벨링 제외.
- 정류장 두 개가 인접해 있으면 각각 별도 bbox.

### 3.4 `roadway`

- 차도면을 외접 bbox로 근사. 4월 단계에서는 segmentation 안 함.
- 횡단보도, 정지선이 보이는 구간은 차도에 포함.
- bbox가 너무 커져 이미지의 50% 이상이 되면, 잘 보이는 차도 영역만 잡고 나머지는 제외.

### 3.5 `sidewalk`

- 차도와 분리 명확한 보도면만 bbox.
- 경계가 모호한 갓길/노견은 보도로 라벨링하지 않는다 (`roadway`로 흡수).

### 3.6 `obstacle`

- 보행 동선상 이동 방해물 — 가로등, 표지판 기둥, 가로수 보호대, 불법 주정차 차량 등.
- 사람/자전거 등 동적 객체는 제외.
- 동일 종류가 여러 개 인접해 있으면 각각 별도 bbox.

### 3.7 `tactile_paving`

- 노란색 점자블록 영역을 외접 bbox.
- 마모/오염되어 노란색이 거의 안 보이는 부분은 라벨링 제외.
- 점형/선형을 분리하지 않고 같은 클래스로 처리.

## 4. 어노테이터 가이드

### 4.1 작업 환경

- 라벨링 도구는 LabelImg / Label Studio / CVAT 중 팀 협의로 결정 (2학기 시작 시점).
- 모니터 해상도 1920×1080 이상 권장.
- 색맹 대응: 점자블록의 노란색 식별이 어려운 경우 어노테이터를 별도 분류.

### 4.2 작업 흐름

```txt
1. 이미지 1장당 약 1~3분 (클래스 평균 3~5개)
2. 작업 50장마다 체크포인트 — 자기 라벨을 다시 검토 (자기 일관성 점검)
3. 작업 200장마다 동료 1인이 상호 검수 (peer review)
4. 미정 케이스 발견 시 즉시 §5 절차
```

### 4.3 작업 외 시간

- 본 가이드와 `class_taxonomy.json`을 작업 시작 전 다시 1회 읽는다.
- 라벨링 도구의 키보드 단축키를 미리 익혀 작업 속도를 일정하게 유지한다.

## 5. 미정 케이스 보고 절차

어노테이터가 다음 경우에 해당하면 자의적으로 판단하지 말고 보고한다.

```txt
- 7개 클래스 중 어느 것에도 잘 들어맞지 않는 객체
- 두 클래스에 모두 해당하는 듯한 객체
- 이미지 전체가 모호하여 5개 이상 라벨링 제외 결정을 한 경우
- 새 클래스가 필요하다고 느껴지는 객체가 5장 이상 누적된 경우
```

보고 방식:

```txt
1. 별도 "미정 케이스" 폴더에 해당 이미지를 표시 (라벨링은 일단 보류)
2. 짧은 메모(1~2 문장)로 어떤 점이 모호한지 기록
3. 김도성 또는 라벨링 책임자가 주 1회 미정 케이스 모음을 검토하여
   - taxonomy 보강 필요한 경우: class_taxonomy.json 갱신 후 어노테이터에게 공유
   - 단발성 케이스: 라벨링 제외 결정으로 정리
```

새 클래스 추가는 어노테이터가 임의로 결정할 수 없다 — 위 절차로 책임자가 공식 결정.

## 6. 어노테이터 간 일치도 (Inter-Annotator Agreement)

### 6.1 IAA 측정 방법

```txt
- 무작위 50장에 대해 어노테이터 2인 이상이 독립 라벨링.
- bbox-level IoU >= 0.5 + class match 인 페어를 일치로 본다.
- 클래스별 정밀도/재현율 형태로 일치도 분포 산출.
```

### 6.2 IAA 임계값

```txt
- 본격 수집 전 시범 IAA가 클래스별 평균 0.7 이상이어야 본격 수집 진입.
- 미달 시 해당 클래스의 라벨링 규칙 재검토 → 본 문서 또는 class_taxonomy.json 보강.
```

### 6.3 4월 단계의 IAA

4월 시범 수집(50~150장) 후 IAA 측정을 1회 수행한다.
4월의 IAA 결과는 본격 수집 진입 결정이 아니라 **본 문서의 미흡한 부분 발견용**이다.

---

## 7. 학습 데이터 빌드 시 일관성 검증

본격 학습 전 데이터셋 빌드 단계에서 자동 검증 스크립트로 다음을 확인한다.
**4월 단계에서는 검증 항목 정의만 하며, 실제 스크립트는 2학기에 작성한다.**

```txt
- 모든 bbox가 이미지 경계 안에 있는가
- bbox의 가로/세로가 16 픽셀 이상인가 (§2.5)
- class_id가 class_taxonomy.json의 7개 id 중 하나와 일치하는가
- 한 이미지 안에 동일 클래스 bbox가 5개 이상이면 검토 대상 표시
- 사람 얼굴/번호판 블러가 적용되었는가 (PII 메타 확인)
- EXIF GPS가 제거 또는 양자화되었는가
```

## 8. 책임자·갱신 절차

- 라벨링 표준 책임자: 김도성 (담당 영역)
- 본 문서는 4월 시범 라벨링 회고 결과 + 2학기 시작 시점 IAA 결과를 반영하여 0.2.x ~
  1.0.0으로 갱신될 수 있다.
- 갱신은 `class_taxonomy.json`의 version 필드와 동기화한다.
